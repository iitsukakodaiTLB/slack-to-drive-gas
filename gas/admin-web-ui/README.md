# admin-web-ui（管理用 GAS Web アプリ）

## Script Properties（必須）

| キー | 説明 |
|------|------|
| `SPREADSHEET_ID` | 設定用スプレッドシート ID（同期ワーカーと同じ） |
| `SLACK_BOT_TOKEN` | 新規チャンネル登録（Slack API）用。Phase 4 以降で使用 |

## 初回セットアップ

1. スクリプトエディタ → **プロジェクトの設定** → **スクリプト プロパティ** に上記を追加
2. **デプロイ** → **新しいデプロイ** → 種類「ウェブアプリ」
   - **次のユーザーとして実行:** 要件により次のいずれか（**監査メールに直結**）
     - **アプリにアクセスしているユーザー** … `Session.getActiveUser().getEmail()` が利用者になりやすい。その場合、スプレッドシートは**当該ユーザーが編集できるよう共有**されている必要があります（`openById` の制約）。
     - **自分** … スプレッドシートはデプロイしたアカウント権限で開ける一方、**アクティブユーザーのメールが空**になりがちです。監査列 `ui_last_updated_by` が空になる可能性があります（README 下記「懸念」を参照）。
   - **アクセスできるユーザー:** 組織内のみ（要件どおり）

## 開発時の注意（懸念・確認事項）

- **監査要件（`ui_last_updated_by`）**と**「シートを利用者に共有しない」**運用は、GAS の「実行ユーザー」設定と**両立が難しい**場合があります。運用方針を決めたうえでデプロイオプションを選んでください。
- 接続確認ボタンで **actorEmail** が空のときは黄色い警告を表示します。本番前に必ず想定どおりのメールが取れるか確認してください。

## clasp

リポジトリルートから:

```bash
npm run clasp:push:admin
```

または:

```bash
cd gas/admin-web-ui && clasp push
```
